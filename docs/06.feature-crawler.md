# 06. 데이터 수집: CU 행사상품 크롤러

## 개요

Python 기반 크롤러로 CU 편의점 공식 쇼핑몰에서 1+1, 2+1 행사상품 정보를 수집하여 CSV로 저장하고, MySQL에 적재한다.

---

## 기능 설명

CU 공식 쇼핑몰(`cu.bgfretail.com`)의 `plusAjax.do` 엔드포인트에 POST 요청을 보내면 서버 측에서 HTML을 렌더링하여 응답한다. 이 HTML을 BeautifulSoup으로 파싱하여 상품명, 가격, 행사종류, 이미지 URL을 추출한다.

### 수집 데이터

| 항목 | 출처 | 예시 |
|------|------|------|
| 상품명 | `div.name p` | 농심)멘토스레인보우 |
| 가격 | `div.price strong` | 900원 |
| 행사종류 | `div.badge span` | 1+1 |
| 이미지 URL | `.prod_img img[src]` | https://...jpg |

### 수집 방식

- `searchCondition=""`: 1+1 행사상품 수집
- `searchCondition="24"`: 2+1 행사상품 수집
- POST 요청 파라미터: `pageIndex=1`, `listType=0`

## 문제 및 해결

### 문제 1: 가격 문자열에 쉼표 포함

CSV에서 가격이 `"4,500"` 형태로 저장되어 있어 정수 변환 시 오류 발생.

```python
# 해결: 쉼표 제거 후 정수 변환
price = int(str(row["가격"]).replace(",", ""))
```

### 문제 2: 이미지 URL이 `//`로 시작하는 경우

일부 이미지 URL이 `//cdn...` 형태의 프로토콜 상대 경로로 제공됨.

```python
# 해결: https: prefix 추가
src = img_elem["src"]
img_url = "https:" + src if src.startswith("//") else src
```

### 문제 3: 상품명에 파일명 사용 불가 문자 포함

이미지 저장 시 상품명을 파일명에 포함하려 하면 `\ / : * ? " < > |` 등의 문자로 인해 오류 발생.

```python
# 해결: 정규식으로 특수문자 제거
clean_name = re.sub(r'[\\/*?:"<>|]', "", prod["상품명"])
```

### 문제 4: 한글 CSV가 Excel에서 깨짐

기본 UTF-8로 저장하면 Excel에서 한글이 깨져 보임.

```python
# 해결: utf-8-sig 인코딩 사용 (BOM 추가)
df.to_csv("cu_products.csv", index=False, encoding="utf-8-sig")
```

## 추가/변경 사항

- `requirements.txt`에는 `selenium`이 포함되어 있지만 실제로는 사용되지 않음 (requests + BeautifulSoup만으로 충분)
- 크롤러는 **페이지 1만 수집** (`pageIndex=1` 고정). 만약 CU 사이트에 상품이 더 많다면 누락될 수 있음
- 이미지 다운로드는 `crawler/images/` 디렉터리에 `인덱스_상품명_파일명` 형식으로 저장
- DB 적재 시 `mart_id=1`로 고정. 이후 `insert_cu_stores.sql`에서 5개 매장으로 재분배됨

# 14. 전체 프로젝트 구조 및 실행 가이드

## 프로젝트 개요

CU 편의점 행사 상품(1+1, 2+1) 정보를 수집하여 지도 기반 웹 UI로 조회하는 풀스택 애플리케이션.

**목적:** 사용자가 원하는 위치를 검색하면 주변 CU 편의점을 지도에서 찾고, 각 지점의 행사상품과 재고를 확인할 수 있음.

## 데이터 흐름

```
[CU 쇼핑몰 웹사이트] ──HTTP POST──▶ cu_crawler.py ──CSV──▶ db_insert.py
                                                               │
                                                               ▼ JDBC
                                                          [MySQL]
                                              database: mart_compare
                                              tables: mart, product, catalog_product
                                                               │
                                                               ▼ JPA
                                              Spring Boot REST API
                                              GET /api/products
                                              GET /api/marts/nearby
                                              POST /api/marts
                                                               │
                                                               ▼ HTTP JSON
React App ──┬── Kakao Map (좌측)
            │   • 검색 위치 마커
            │   • CU 매장 마커들 (Kakao Places API)
            │   • 호버 InfoWindow / 클릭 → 상품 로딩
            │
            └── Sidebar (우측)
                • "주변 CU 편의점 N곳"
                • 선택된 매장명
                • [전체] [1+1] [2+1] 필터 탭
                • 상품 리스트 (이미지 + 이름 + 가격 + 행사종류 + 재고)
```

## API 엔드포인트

| Method | Path | 설명 |
|--------|------|------|
| GET | `/api/products` | 전체 상품 조회 |
| GET | `/api/products?martId=X` | 특정 매장의 상품 조회 |
| GET | `/api/marts/nearby?lat=&lng=&radius=` | 주변 CU 매장 검색 |
| POST | `/api/marts` | 매장 찾기 또는 생성 (find-or-create) |

## 실행 방법

```bash
# 1. 데이터베이스 설정
mysql -u root -p < "mysql/mart, product table create.sql"
mysql -u root -p mart_compare -e "INSERT INTO mart (name) VALUES ('CU');"
mysql -u root -p mart_compare < mysql/insert_cu_stores.sql
mysql -u root -p mart_compare < mysql/add_stock_column.sql
mysql -u root -p mart_compare < mysql/catalog_and_redistribute.sql

# 2. 크롤링 (선택사항, CSV/이미지가 이미 있으면 생략)
cd crawler
pip install -r requirements.txt
python cu_crawler.py
python db_insert.py
cd ..

# 3. 백엔드 실행
cd backend && ./gradlew bootRun

# 4. 프론트엔드 실행 (다른 터미널)
cd frontend && npm run dev
```

브라우저에서 `http://localhost:5173` 접속.

## 알려진 이슈

### 버그
1. **insert_cu_stores.sql 중복** — UPDATE 문이 2번 작성됨 (무해함)
2. **최초 검색 시 중복 API 호출** — 첫 번째 CU 자동 선택 + 동일 마커 클릭 이벤트가 중복 실행됨
3. **Zone.Identifier 파일** — Windows에서 넘어온 ADS 파일 13개 (backend 디렉터리)

### 한계
1. **재고는 랜덤** — 실제 CU 재고 데이터가 아님 (외부 API 미제공)
2. **크롤러 페이지 1만 수집** — CU 사이트에 더 많은 상품이 있어도 누락
3. **단일 컴포넌트** — App.tsx에 모든 로직 집중 (분할 필요)
4. **DTO/Valid 없음** — 엔티티 직접 반환, 입력 검증 없음
5. **No Docker** — 모든 서비스 로컬에서 수동 실행
6. **CORS localhost 한정** — 운영 배포 시 수정 필요

# Phase 4: 주변 CU 편의점 실시간 검색 및 마커 표시 (2026-05-27)

## 작업 내용

위치 검색 시 Kakao Places API로 주변 CU 편의점을 실시간 조회하여 지도에 마커로 표시하고, 개수를 출력하며, 마커 클릭 시 행사상품을 보여주는 기능 구현.

---

## 작업 순서

### 1. 주변 CU 편의점 개수 표시 추가

**파일:** `frontend/src/App.tsx`

사이드바 상단에 `marts.length`를 읽어 "주변 CU 편의점 N곳" 텍스트를 출력. 검색 결과가 있을 때만 노출.

```tsx
{marts.length > 0 && (
  <div style={{ fontSize: '13px', color: '#666', marginBottom: '8px' }}>
    주변 CU 편의점 {marts.length}곳
  </div>
)}
```

---

### 2. fetch 에러 핸들링 및 검색 위치 마커 추가

처음에는 검색해도 마커와 개수가 보이지 않는 문제가 있었음. 원인은 두 가지:

1. **fetch에 `.catch()`가 없음** → 백엔드가 꺼져 있거나 오류가 나도 아무 일도 일어나지 않음
2. **검색 위치 마커가 없음** → 지도가 이동만 할 뿐 시각적 피드백이 없음

**수정:**
- `clearMarkers()`를 fetch 전으로 이동, 검색 위치에 기본 마커 추가
- fetch 응답 코드 체크(`res.ok`) 및 `.catch()`로 오류 처리
- 오류 시 `setMarts([])`, `setProducts([])`로 UI 정상 상태 유지

---

| CU 광화문광장점 | 서울 종로구 새문안로 103-1 | 37.5710, 126.9766 |
| CU 명동역점 | 서울 중구 퇴계로 127 | 37.5608, 126.9860 |
| CU 을지로3가점 | 서울 중구 충무로 53-1 | 37.5662, 126.9820 |
| CU 종각쇼핑센터점 | 서울 종로구 종로 지하 73 | 37.5700, 126.9832 |

### 4. Kakao Places API로 CU 실시간 검색으로 전환

**문제:** DB에 5개만 박아두면 "서울시청" 근처에서만 동작하고 다른 위치 검색 시 무용지물. 또한 Kakao Places 결과와 DB 좌표가 달라 매칭이 안 됨.

**해결:** DB 기반 검색을 폐기하고 **Kakao Places API `keywordSearch('CU')`** 로 실시간 검색

**변경된 데이터 흐름:**

```
검색창 입력
  → KakaoMap Places.keywordSearch(키워드) → lat/lng 획득
  → 지도 이동 + 검색 위치 마커 표시
  → KakaoMap Places.keywordSearch('CU', { location, radius: 1000 }) → 주변 CU 목록
  → 지도에 CU 마커들 표시 (InfoWindow 포함)
  → 사이드바에 "주변 CU 편의점 N곳" 표시
  → 첫 번째 CU 자동 선택 + GET /api/products → 전체 행사상품 로딩
  → 마커 클릭 → setSelectedCu() + GET /api/products → 상품 목록 갱신
```

**핵심 변경:**
- `addCUMarkers()` 함수 제거 (데드코드)
- `selectMart()` / `selectedMartId` 제거 (더 이상 DB mart_id로 상품 조회 안 함)
- `selectedCu` 상태 추가 (현재 선택된 Kakao CU 매장)
- `selectCuWithProducts()` → `GET /api/products`로 **모든 상품** 조회 (CU 행사는 전국 동일)
- 불필요한 `KakaoPlace` 인터페이스 제거

---

### 5. 매장별 재고(stock) 표시 추가

**문제:** Kakao 마커로 CU 매장을 찾을 수 있지만, 해당 지점에 상품이 실제로 몇 개나 있는지 알 수 없음.

**DB — `mysql/add_stock_column.sql`**
```sql
alter table product add column stock int not null default 0;
update product set stock = floor(3 + rand() * 48);
```
각 상품에 3~50개 사이의 랜덤 재고값 부여.

**Backend — `entity/Product.java`**
- `private Integer stock;` 필드 추가 (JPA가 자동 매핑)

**Frontend — `App.tsx`**
- `selectCuWithProducts(lat, lng)`가 좌표를 받아 주변 200m 내 DB 매장을 찾음
- DB 매장이 있으면 → `GET /api/products?martId=X` → 해당 매장의 상품 + 재고 로딩
- 없으면 → `GET /api/products` → 전체 상품 로딩 (fallback)
- 상품 카드에 "재고 N개" 표시 (10개 초과 초록, 10개 이하 빨강)

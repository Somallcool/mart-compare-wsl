# 동적 매장 생성 및 지점별 상품·재고 표시

## 작업 내용

Kakao Places API로 찾은 CU 편의점(DB에 없는 지점)을 클릭했을 때, 해당 지점의 행사상품과 재고를 표시하는 기능 구현.

---

## 문제 상황

기존 구조의 한계:

1. **DB에는 5개 CU 매장만 존재** (서울 시청 인근) → 수원역, 강남 등 다른 지역 검색 시 매장을 찾을 수 없음
2. **마커는 Kakao Places로 실시간 검색**하지만, DB 매장과 ID가 일치하지 않아 상품 연결 불가
3. **재고(stock) 컬럼은 존재**하지만 모든 매장이 공유하는 랜덤 값 → 지점별 재고 개념 없음

---

## 해결 방법

### 1. find-or-create API 추가

**파일:** `backend/.../controller/MartController.java`

```java
@PostMapping
public Mart findOrCreate(@RequestBody Map<String, Object> body) {
    String name = (String) body.get("name");
    String address = (String) body.get("address");
    double lat = ((Number) body.get("latitude")).doubleValue();
    double lng = ((Number) body.get("longitude")).doubleValue();
    return martService.findOrCreate(name, address, lat, lng);
}
```

**파일:** `backend/.../service/MartService.java`

```java
@Transactional
public Mart findOrCreate(String name, String address, double lat, double lng) {
    // 100m 이내 기존 매장이 있으면 반환
    List<Mart> nearby = martRepository.findNearby(lat, lng, 0.1);
    if (!nearby.isEmpty()) {
        return nearby.get(0);
    }

    // 없으면 새 매장 생성
    Mart mart = new Mart();
    mart.setName(name);
    mart.setAddress(address);
    mart.setLatitude(lat);
    mart.setLongitude(lng);
    mart = martRepository.save(mart);

    // 카탈로그의 모든 80개 상품을 이 매장용으로 복제 + 랜덤 재고 할당
    List<CatalogProduct> catalog = catalogProductRepository.findAll();
    for (CatalogProduct cp : catalog) {
        Product product = new Product();
        product.setMartId(mart.getId());
        product.setName(cp.getName());
        product.setPrice(cp.getPrice());
        product.setEventType(cp.getEventType());
        product.setImageUrl(cp.getImageUrl());
        product.setStock(3 + (int) (Math.random() * 48)); // 3~50
        productRepository.save(product);
    }

    return mart;
}
```

### 2. 모든 매장을 찾는 Native Query 추가

기존 `findNearbyCUs`는 `WHERE m.name = 'CU'` 조건이 있어서 동적으로 생성된 매장(이름이 "CU 수원역점" 등)을 찾을 수 없었다. 이름 조건 없이 위도/경도로만 검색하는 쿼리를 추가했다.

**파일:** `backend/.../repository/MartRepository.java`

```java
@Query(value = """
    SELECT m.*,
        (6371 * acos(cos(radians(:lat)) * cos(radians(m.latitude))
        * cos(radians(m.longitude) - radians(:lng))
        + sin(radians(:lat)) * sin(radians(m.latitude)))) AS distance
    FROM mart m
    HAVING distance <= :radius
    ORDER BY distance
    """, nativeQuery = true)
List<Mart> findNearby(@Param("lat") double lat, @Param("lng") double lng, @Param("radius") double radius);
```

### 3. @Setter 추가

새로운 Mart/Product 객체를 생성할 때 필드 값을 설정해야 하므로, 기존 `@Getter`만 있던 엔티티에 `@Setter`를 추가했다.

**파일:** `backend/.../entity/Mart.java`, `backend/.../entity/Product.java`

```java
@Getter
@Setter   // ← 추가
@Entity

```

### 4. 프론트엔드 수정

**파일:** `frontend/src/App.tsx`

마커 클릭 시 `POST /api/marts` (find-or-create) → `GET /api/products?martId=...` 순서로 호출한다.

```typescript
const selectCuWithProducts = (cu: Mart) => {
  setSelectedCu(cu)
  fetch(`http://localhost:8081/api/marts`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      name: cu.name,
      address: cu.address,
      latitude: cu.latitude,
      longitude: cu.longitude
    })
  })
    .then(res => res.json())
    .then((mart: Mart) => fetch(`http://localhost:8081/api/products?martId=${mart.id}`))
    .then(res => res.json())
    .then((data: Product[]) => {
      if (Array.isArray(data)) setProducts(data)
      else setProducts([])
    })
    .catch(() => setProducts([]))
}
```

## 데이터 흐름

```
사용자 검색 "수원역"
  → Kakao Places.keywordSearch('수원역') → lat/lng
  → Kakao Places.keywordSearch('CU', { location, radius: 1000 }) → CU 목록
  → 지도에 마커 표시
  → 첫 번째 CU 자동 선택

마커 클릭
  → POST /api/marts { name, address, latitude, longitude }
    → 100m 이내 기존 매장? → 반환
    → 없음? → 새 매장 생성 + catalog 80개 복제 + 랜덤 재고
  → GET /api/products?martId={id}
    → 해당 매장의 80개 상품 + 재고 반환
  → 우측 사이드바에 표시 (1+1 / 2+1 필터 가능)
```

## 주의사항

- 재고는 `Math.random() * 48 + 3`으로 **랜덤 생성**된 값입니다. 실제 CU 재고 데이터가 아닙니다.
- CU의 실시간 재고는 외부에 공개되는 API가 없어 가져올 수 없습니다.
- 같은 위치를 다시 클릭하면 100m 이내 기존 매장을 찾아 재사용하므로 중복 생성되지 않습니다.
- `ddl-auto=none`이므로 `catalog_product` 테이블은 SQL로 직접 생성해야 합니다.

# 10. 주소 검색 및 주변 CU 매장 마커 표시

## 개요

사용자가 주소나 장소명을 검색하면 Kakao Places API로 위치를 찾고, 해당 위치 주변의 CU 편의점들을 지도에 마커로 표시한다.

---

## 전체 흐름

```
사용자 입력 "수원역"
    │
    ▼
Kakao Places.keywordSearch("수원역")
    │ latitude=37.2662, longitude=126.9995
    ▼
검색 위치로 지도 이동 + 검색 마커 표시
    │
    ▼
Kakao Places.keywordSearch("CU", { location, radius: 1000 })
    │ 주변 CU 편의점 목록 (실시간)
    ▼
각 CU에 마커 생성 + InfoWindow(주소) + 클릭 이벤트
    │
    ▼
첫 번째 CU 자동 선택 → 상품 로딩
```

---

## 구현 상세

### Kakao Places 키워드 검색

```typescript
const places = new window.kakao.maps.services.Places()

// 1단계: 사용자 입력 위치 검색
places.keywordSearch(keyword, (results, status) => {
    if (status !== window.kakao.maps.services.Status.OK) return
    
    const lat = parseFloat(results[0].y)
    const lng = parseFloat(results[0].x)
    map.setCenter(new window.kakao.maps.LatLng(lat, lng))
    
    // 2단계: 주변 CU 매장 검색
    places.keywordSearch('CU', (cuResults, cuStatus) => {
        // CU 매장 목록 처리
    }, {
        location: center,    // 검색 위치 기준
        radius: 1000          // 반경 1km
    })
})
```

- Kakao Places API는 DB와 무관하게 **실시간**으로 CU 매장을 검색
- 검색 반경 1km, 거리순 정렬
- 결과를 `Mart[]` 타입으로 매핑 (id는 배열 인덱스, 실제 DB ID와 무관)

### 마커 인터랙션

```typescript
// 마커 생성
const marker = new window.kakao.maps.Marker({
    position: pos,
    map,
    title: cu.name
})

// 호버 시 InfoWindow (주소 표시)
window.kakao.maps.event.addListener(marker, 'mouseover', () => infowindow.open(map, marker))
window.kakao.maps.event.addListener(marker, 'mouseout', () => infowindow.close())

// 클릭 시 해당 매장 상품 로딩
window.kakao.maps.event.addListener(marker, 'click', () => {
    selectCuWithProducts(cu)
})
```

## 처리 과정 및 변경 이력

### Phase 1: DB 기반 검색 (02.check-projects.md)

처음에는 백엔드의 `GET /api/marts/nearby`로 DB에 저장된 5개 CU 매장을 검색했다.

**문제:** DB에 있는 5개 매장(서울 시청 인근) 외의 지역에서는 검색 결과가 없음.

```typescript
// 초기 방식: DB 기반 검색 (폐기됨)
fetch(`http://localhost:8081/api/marts/nearby?lat=${lat}&lng=${lng}&radius=1`)
```

### Phase 2: Kakao Places 실시간 검색 (03.check-projects.md)

DB 검색을 폐기하고 Kakao Places API `keywordSearch('CU')`로 전환.

**변경 사항:**
- `addCUMarkers()` 함수 제거 (데드코드)
- `selectedMartId` / `selectMart()` 제거
- `selectedCu` 상태 추가 (선택된 Kakao CU 매장)
- `Mart[]` 인터페이스 재사용 (id는 Kakao 배열 인덱스)

**새로운 문제:** Kakao CU 매장과 DB 매장의 ID가 달라 상품 연결 불가

### Phase 3: find-or-create + 동적 매장 생성 (05.dynamic-mart-creation.md)

Kakao 매장 클릭 시 DB에 매장을 생성(find-or-create)하고 상품을 할당.

```typescript
// 최종 방식: find-or-create
fetch(`http://localhost:8081/api/marts`, {
    method: 'POST',
    body: JSON.stringify({ name, address, latitude, longitude })
})
    .then(res => res.json())
    .then(mart => fetch(`http://localhost:8081/api/products?martId=${mart.id}`))
```

## 문제 및 해결

### 문제 1: Kakao Places 결과와 DB 매장 불일치

Kakao에서 가져온 CU 매장의 ID(배열 인덱스)는 DB의 mart_id(1~5)와 전혀 다름. 마커 클릭 시 어떤 상품을 보여줘야 할지 알 수 없었음.

**1차 해결 (03):** 마커 클릭 시 `GET /api/products`로 전체 상품 조회 (지점 무관)
**2차 해결 (05):** `POST /api/marts` find-or-create로 Kakao 매장을 DB에 등록 후 상품 조회

### 문제 2: fetch 에러 핸들링 부재

초기에는 `.catch()`가 없어 API 오류 시 UI가 멈춤.

```typescript
// 해결: catch 추가
.catch(() => {
    setMarts([])
    setProducts([])
})
```

### 문제 3: 검색 위치 마커 없음

지도만 이동할 뿐 시각적 피드백이 없어 검색이 되었는지 알기 어려웠음.

```typescript
// 해결: 검색 위치에 마커 추가
const searchMarker = new window.kakao.maps.Marker({
    position: center,
    map,
    title: keyword
})
```

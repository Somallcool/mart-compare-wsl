# 12. 재고 시스템 (Stock)

## 개요

상품에 재고(stock) 개념을 도입하여 각 매장별 상품의 재고 수량을 표시한다. 단, 실제 CU 재고 데이터가 아닌 데모 목적의 랜덤 값이다.

---

## 데이터베이스

### stock 컬럼 추가

```sql
ALTER TABLE product ADD COLUMN stock INT NOT NULL DEFAULT 0;
```

### 랜덤 재고 할당 (3~50)

```sql
UPDATE product SET stock = FLOOR(3 + RAND() * 48);
```

### 동적 생성 매장의 재고

새 매장이 생성될 때 `MartService.findOrCreate()`에서 카탈로그 상품을 복제하며 각각 랜덤 재고를 할당한다.

```java
product.setStock(3 + (int) (Math.random() * 48));
```

## 프론트엔드

### 재고 표시

```tsx
{p.stock != null && (
    <div style={{ fontSize: '12px', color: p.stock > 10 ? '#2e7d32' : '#d32f2f' }}>
        재고 {p.stock}개
    </div>
)}
```

- 재고가 `null`이면 표시하지 않음
- 10개 초과: 초록색 (`#2e7d32`)
- 10개 이하: 빨간색 (`#d32f2f`)

## 실제 재고 데이터 vs 데모 데이터

**현재 재고는 모두 랜덤 생성된 가상의 데이터입니다.**

| 구분 | 실제 재고 | 현재 방식 |
|------|:---------:|:---------:|
| 데이터 출처 | CU POS 시스템 | `Math.random() * 48 + 3` |
| 매장별 차이 | 실제 재고 상황 반영 | 랜덤 (매장/상품마다 다름) |
| 실시간성 | 실시간 변동 | 고정 (한 번 할당되면 변경 안 됨) |
| 획득 가능성 | ❌ 외부 API 미제공 | — |

CU 편의점을 포함한 국내 편의점들은 실시간 재고를 외부에 제공하는 공개 API가 없습니다. 재고 데이터는 각 매장의 POS 시스템 내에서만 관리되며, 외부에서 접근할 수 없습니다.

## 문제 및 해결

### 문제 1: 재고가 없던 기존 상품

초기 DB에는 `stock` 컬럼 자체가 없었음. 기존 데이터(80개 상품)에 대해 일괄 추가 필요.

```sql
-- 해결: ALTER TABLE + UPDATE
ALTER TABLE product ADD COLUMN stock INT NOT NULL DEFAULT 0;
UPDATE product SET stock = FLOOR(3 + RAND() * 48);
```

### 문제 2: 동적 생성 매장의 재고

새로운 매장을 생성할 때도 재고가 필요. 매번 랜덤 값을 할당.

```java
// MartService.findOrCreate()에서 상품 복제 시
product.setStock(3 + (int) (Math.random() * 48));
```

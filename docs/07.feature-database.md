# 07. 데이터베이스 설계 및 관리

## 개요

MySQL 8을 사용하며, JPA의 `ddl-auto=none` 전략으로 수동 SQL로 스키마를 관리한다. UTF-8MB4 인코딩 사용.

---

## 테이블 구조

### mart — 편의점 매장

```sql
CREATE TABLE mart (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50) NOT NULL,       -- 'CU' / Kakao Places 매장명
    address VARCHAR(200),             -- 주소
    latitude DOUBLE,                  -- 위도
    longitude DOUBLE                  -- 경도
);
```

### product — 매장별 상품 (재고 포함)

```sql
CREATE TABLE product (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    mart_id BIGINT NOT NULL,           -- FK → mart.id
    name VARCHAR(100) NOT NULL,        -- 상품명
    price INT NOT NULL,                -- 가격
    event_type VARCHAR(10) NOT NULL,   -- '1+1' or '2+1'
    image_url VARCHAR(500),            -- 이미지 URL
    stock INT NOT NULL DEFAULT 0,      -- 재고 수량 (나중에 추가됨)
    FOREIGN KEY (mart_id) REFERENCES mart (id)
);
```

### catalog_product — 상품 카탈로그 (마스터)

```sql
CREATE TABLE catalog_product (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    price INT NOT NULL,
    event_type VARCHAR(10) NOT NULL,
    image_url VARCHAR(500)
);
```

> **catalog_product**는 80개 상품의 중복 제거된 마스터 테이블. 동적 매장 생성 시 이 테이블을 기준으로 상품을 복제한다.

---

## 관련 파일

| 파일 | 역할 |
|------|------|
| `mysql/mart, product table create.sql` | 초기 스키마 생성 (mart + product) |
| `mysql/insert_cu_stores.sql` | CU 매장 5개 시드 데이터 + 상품 분배 |
| `mysql/add_stock_column.sql` | stock 컬럼 추가 + 랜덤 재고 할당 |
| `mysql/catalog_and_redistribute.sql` | catalog_product 생성 + 상품 재분배 |

---

## 마이그레이션 순서

```bash
# 1. 데이터베이스 및 테이블 생성
mysql -u root -p < "mysql/mart, product table create.sql"

# 2. 초기 매장 데이터 INSERT (mart 1개, 좌표 없음)
mysql -u root -p mart_compare -e "INSERT INTO mart (name) VALUES ('CU');"

# 3. 크롤링한 상품 데이터 적재 (crawler/db_insert.py 실행 후)

# 4. CU 매장 5개 좌표 업데이트 + 상품 분배
mysql -u root -p mart_compare < mysql/insert_cu_stores.sql

# 5. 재고 컬럼 추가
mysql -u root -p mart_compare < mysql/add_stock_column.sql

# 6. 카탈로그 테이블 생성 + 상품 재분배 (2+1 버그 수정)
mysql -u root -p mart_compare < mysql/catalog_and_redistribute.sql
```

## 문제 및 해결

### 문제 1: Zone.Identifier 파일 생성

Windows에서 파일을 내려받으면 `filename:Zone.Identifier` 형태의 NTFS ADS(Alternate Data Stream) 파일이 함께 생성됨. 총 13개 파일이 backend 디렉터리에 존재.

```bash
# 확인
find backend -name "*:Zone.Identifier"
# 삭제
find backend -name "*:Zone.Identifier" -delete
```

### 문제 2: ddl-auto=none 관리

`spring.jpa.hibernate.ddl-auto=none`으로 설정되어 있어 JPA가 테이블을 자동 생성하지 않음. 새 엔티티(CatalogProduct)를 추가할 때마다 수동으로 SQL을 실행해야 함.

```properties
# application.properties
spring.jpa.hibernate.ddl-auto=none
```

## 추가/변경 사항

- 초기에는 `stock` 컬럼이 없었으나 `add_stock_column.sql`로 추가됨
- `catalog_product` 테이블은 2+1 버그 수정 과정에서 추가됨
- `insert_cu_stores.sql`에 UPDATE 문이 중복으로 작성되어 있음 (2회 실행되지만 결과는 동일)
- DB 계정 정보(root/1234)는 개발 환경용 하드코딩. 실제 배포 시 분리 필요

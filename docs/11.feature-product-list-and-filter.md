# 11. 상품 리스트 및 1+1 / 2+1 필터

## 개요

우측 사이드바에 상품 리스트를 표시하고, 행사 종류(1+1 / 2+1)별로 필터링할 수 있는 탭 기능을 제공한다.

---

## 상품 리스트 UI

```tsx
{filtered.map(p => (
    <div key={p.id} style={{ borderBottom: '1px solid #eee', padding: '8px 0', display: 'flex', gap: '12px' }}>
        {p.imageUrl && (
            <img src={p.imageUrl} alt={p.name}
                style={{ width: '80px', height: '80px', objectFit: 'contain', flexShrink: 0 }}
                onError={e => { (e.target as HTMLImageElement).style.display = 'none' }}
            />
        )}
        <div>
            <div style={{ fontWeight: 'bold' }}>{p.name}</div>
            <div>{p.price.toLocaleString()}원</div>
            <div style={{ color: 'red', fontSize: '12px' }}>{p.eventType}</div>
            {p.stock != null && (
                <div style={{ fontSize: '12px', color: p.stock > 10 ? '#2e7d32' : '#d32f2f' }}>
                    재고 {p.stock}개
                </div>
            )}
        </div>
    </div>
))}
```

- 이미지: 80x80, 가운데 정렬, 로드 실패 시 숨김
- 상품명: 굵은 글씨
- 가격: `toLocaleString()`으로 3자리마다 쉼표
- 행사종류: 빨간색, 12px
- 재고: 10개 초과 → 초록, 10개 이하 → 빨강

## 필터 탭

```typescript
type Tab = 'all' | '1+1' | '2+1'
const [activeTab, setActiveTab] = useState<Tab>('all')

// 필터링 (클라이언트 측)
const filtered = activeTab === 'all'
    ? products
    : products.filter(p => p.eventType === activeTab)
```

```tsx
{(['all', '1+1', '2+1'] as Tab[]).map(tab => (
    <button key={tab} onClick={() => setActiveTab(tab)}
        style={{
            flex: 1, padding: '8px', border: '1px solid #ccc', borderRadius: '6px',
            cursor: 'pointer', fontWeight: activeTab === tab ? 'bold' : 'normal',
            background: activeTab === tab ? '#ffeb3b' : '#fff'
        }}
    >
        {tab === 'all' ? '전체' : tab}
    </button>
))}
```

- 3개 버튼 `flex: 1`로 균등 배치
- 활성 탭: 노란 배경(`#ffeb3b`) + 굵은 글씨
- 비활성 탭: 흰 배경 + 일반 글씨

## 문제 및 해결

### 문제 1: 2+1 탭에 아무것도 안 나옴

**원인:** 상품 분배 방식이 ID 범위 기반이라 매장마다 한 가지 행사 유형만 있었음.

| 매장 | 보유 상품 (초기) | 1+1 | 2+1 |
|------|-----------------|:---:|:---:|
| mart 1 | ID 1~16 | 16개 | 0개 |
| mart 2 | ID 17~32 | 16개 | 0개 |
| mart 3 | ID 33~48 | 0개 | 16개 |
| mart 4 | ID 49~64 | 0개 | 16개 |
| mart 5 | ID 65~80 | 0개 | 16개 |

**해결:** 모듈로 연산으로 재분배하여 각 매장이 1+1 8개 + 2+1 8개씩 보유하도록 수정.

```sql
UPDATE product SET mart_id = ((id - 1) % 5) + 1;
```

자세한 내용은 `04.product-redistribution-and-2plus1-fix.md` 참조.

### 문제 2: 가격 포맷팅

가격이 숫자로 저장되어 있어 `4500`을 `4,500원`으로 표시해야 함.

// 해결: toLocaleString()

## 추가/변경 사항

- **초기**: 필터 없이 전체 상품 리스트만 표시
- **01.check-projects.md**: `Tab` 타입, `activeTab` 상태, 필터링 로직, 탭 UI 추가
- **서버 필터링 vs 클라이언트 필터링**: 현재 데이터가 80개로 적어 클라이언트 측 `Array.filter()` 사용. 데이터가 많아지면 `?eventType=1+1` 쿼리 파라미터로 서버 측 필터링 고려
- `filtered`는 파생 변수로 `products` 원본 배열 불변성 유지
- 상품 개수 표시도 `products.length` → `filtered.length`로 변경

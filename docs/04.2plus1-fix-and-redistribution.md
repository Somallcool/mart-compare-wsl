# 1+1 / 2+1 혼합 분배 및 2+1 출력 안 되던 문제 수정

## 작업 내용

2+1 탭을 선택해도 아무 상품도 표시되지 않는 버그 수정 및 상품 분배 방식 개선.

---

## 문제 원인

### 상품 분배 방식의 문제

기존에는 `insert_cu_stores.sql`에서 **ID 범위 기반**으로 상품을 매장에 분배했다.

```sql
update product set mart_id = 1 where id between 1 and 16;
update product set mart_id = 2 where id between 17 and 32;
update product set mart_id = 3 where id between 33 and 48;
update product set mart_id = 4 where id between 49 and 64;
update product set mart_id = 5 where id between 65 and 80;
```

CSV 데이터는 **1+1 상품 40개(1~40번) → 2+1 상품 40개(41~80번)** 순서로 저장되어 있어서, ID 범위 분배 시 각 매장이 한 가지 행사 유형만 갖게 되는 문제가 발생했다.

| 매장 | 보유 상품 | 1+1 | 2+1 |
|------|----------|:---:|:---:|
| mart 1 (서울시청점) | ID 1~16 | 16개 | 0개 |
| mart 2 (광화문광장점) | ID 17~32 | 16개 | 0개 |
| mart 3 (명동역점) | ID 33~48 | 0개 | 16개 |
| mart 4 (을지로3가점) | ID 49~64 | 0개 | 16개 |
| mart 5 (종각점) | ID 65~80 | 0개 | 16개 |

→ 사용자가 1+1만 있는 매장(mart 1, 2)에서 2+1 탭을 누르면 빈 화면. 반대도 동일.

---

## 해결 방법

### 카탈로그 테이블 생성

각 매장이 전체 상품을 참조할 수 있도록 `catalog_product` 테이블을 생성하고 기존 80개 상품을 중복 제거하여 삽입했다.

**파일:** `mysql/catalog_and_redistribute.sql`

```sql
CREATE TABLE IF NOT EXISTS catalog_product (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    price INT NOT NULL,
    event_type VARCHAR(10) NOT NULL,
    image_url VARCHAR(500)
);

INSERT INTO catalog_product (name, price, event_type, image_url)
SELECT DISTINCT name, price, event_type, image_url FROM product;
```

### 모듈로 연산으로 재분배

```sql
UPDATE product SET mart_id = ((id - 1) % 5) + 1;
```

- ID 1 → mart 1, ID 2 → mart 2, ..., ID 5 → mart 5, ID 6 → mart 1 (반복)
- 1+1 상품 40개(ID 1~40)가 5개 매장에 8개씩 분배
- 2+1 상품 40개(ID 41~80)가 5개 매장에 8개씩 분배

**결과:** 각 매장이 1+1 **8개** + 2+1 **8개** = 총 16개씩 보유

# 1+1 / 2+1 필터 버튼 추가

## 작업 내용

상품 리스트에 **전체 / 1+1 / 2+1** 탭 버튼을 추가하여 행사 종류별로 상품을 필터링할 수 있도록 개선.

## 변경된 파일

- `frontend/src/App.tsx`

## 변경 사항 상세

### 1. 타입 및 상태 추가 (`App.tsx:18,23`)

```typescript
type Tab = 'all' | '1+1' | '2+1'

const [activeTab, setActiveTab] = useState<Tab>('all')
```

- `Tab` 유니언 타입으로 탭 값을 제한하여 타입 안전성 확보
- `activeTab` 상태로 현재 선택된 탭 관리 (기본값: `'all'`)

### 2. 필터링 로직 (`App.tsx:73-75`)

```typescript
const filtered = activeTab === 'all'
  ? products
  : products.filter(p => p.eventType === activeTab)
```

- `products` 원본 배열은 유지하고, `filtered`로 파생 변수를 계산
- `activeTab`이 `'all'`이면 전체, 아니면 `eventType`이 일치하는 상품만 필터링

### 3. 탭 버튼 UI (`App.tsx:85-103`)

```typescript
{(['all', '1+1', '2+1'] as Tab[]).map(tab => (
  <button
    key={tab}
    onClick={() => setActiveTab(tab)}
    style={{
      flex: 1,
      padding: '8px',
      border: '1px solid #ccc',
      borderRadius: '6px',
      cursor: 'pointer',
      fontWeight: activeTab === tab ? 'bold' : 'normal',
      backgroundColor: activeTab === tab ? '#ffeb3b' : '#fff',
    }}
  >
    {tab === 'all' ? '전체' : tab}
  </button>
))}
```

- 3개 버튼을 `flex`로 균등 배치
- 활성화된 탭은 **노란 배경(`#ffeb3b`) + 굵은 글씨**로 시각적 구분
- `상품 개수 표시`도 `products.length`에서 `filtered.length`로 변경하여 현재 탭에 맞는 개수를 보여줌

## 접근 방식

- **서버 필터링 x, 클라이언트 필터링:** 현재 데이터 규모(80개)가 작아 클라이언트 측 `Array.filter()`로 처리. 추천 데이터가 많아지면 서버 측 쿼리 파라미터(`?eventType=1+1`)로 전환 가능
- **원본 배열 불변성 유지:** `filter()`는 새 배열을 반환하므로 `products` 상태는 그대로 보존
- **타입 안전성:** `Tab` 타입과 `as Tab[]` 단언으로 버튼 값 오타 방지

## 문제 및 해결

| 문제 | 해결 |
|------|------|
| `activeTab` 초기값이 `'all'`이므로 최초 렌더링 시 전체 상품 표시 | 의도한 동작, 별도 처리 불필요 |
| 버튼 클릭 시 리스트만 필터링되고 지도는 그대로 | 지도는 모든 마커(현재 1개)를 계속 표시하도록 유지, 필터와 독립적으로 동작 |

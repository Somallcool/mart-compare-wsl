# 13. 동적 매장 생성 (find-or-create)

## 개요

Kakao Places API로 찾은 CU 편의점(DB에 없는 지점)을 클릭했을 때, 해당 지점을 DB에 생성하고 행사상품을 할당하여 지점별 상품 조회를 가능하게 한다.

---

## 배경

### 문제 상황

1. Kakao Places API는 전국 모든 CU 매장을 실시간으로 찾아 마커로 표시
2. 하지만 DB에는 서울 시청 인근 5개 매장만 존재
3. Kakao 매장 ID(배열 인덱스)와 DB 매장 ID(1~5)가 전혀 다름
4. 따라서 마커를 클릭해도 해당 지점의 상품을 조회할 방법이 없었음

### 해결 방향

- 마커 클릭 시 해당 위치의 Kakao 매장 정보(lat, lng, name, address)를 백엔드로 전송
- 백엔드는 100m 이내에 기존 매장이 있는지 확인
- 없으면 새 매장을 생성하고, catalog_product의 80개 상품을 복제 + 랜덤 재고 할당
- 생성된 DB 매장 ID로 상품 조회

---

## 구현

### 백엔드: MartService.findOrCreate()

```java
@Transactional
public Mart findOrCreate(String name, String address, double lat, double lng) {
    // 1. 100m 이내 기존 매장 확인
    List<Mart> nearby = martRepository.findNearby(lat, lng, 0.1);
    if (!nearby.isEmpty()) {
        return nearby.get(0);  // 기존 매장 반환
    }

    // 2. 새 매장 생성
    Mart mart = new Mart();
    mart.setName(name);
    mart.setAddress(address);
    mart.setLatitude(lat);
    mart.setLongitude(lng);
    mart = martRepository.save(mart);

    // 3. 카탈로그 상품 80개 복제 + 랜덤 재고
    List<CatalogProduct> catalog = catalogProductRepository.findAll();
    for (CatalogProduct cp : catalog) {
        Product product = new Product();
        product.setMartId(mart.getId());
        product.setName(cp.getName());
        product.setPrice(cp.getPrice());
        product.setEventType(cp.getEventType());
        product.setImageUrl(cp.getImageUrl());
        product.setStock(3 + (int) (Math.random() * 48));
        productRepository.save(product);
    }

    return mart;
}
```

### 백엔드: MartController (POST /api/marts)

```java
@PostMapping
public Mart findOrCreate(@RequestBody Map<String, Object> body) {
    String name = (String) body.get("name");
    String address = (String) body.get("address");
    double lat = ((Number) body.get("latitude")).doubleValue();
    double lng = ((Number) body.get("longitude")).doubleValue();
    return martService.findOrCreate(name, address, lat, lng);
}
```

### Repository: findNearby 쿼리

```java
@Query(value = """
    SELECT m.*,
        (6371 * acos(cos(radians(:lat)) * cos(radians(m.latitude))
        * cos(radians(m.longitude) - radians(:lng))
        + sin(radians(:lat)) * sin(radians(m.latitude)))) AS distance
    FROM mart m
    HAVING distance <= :radius
    ORDER BY distance
    """, nativeQuery = true)
List<Mart> findNearby(@Param("lat") double lat, @Param("lng") double lng, @Param("radius") double radius);
```

> 기존 `findNearbyCUs`는 `WHERE m.name = 'CU'` 조건이 있어 동적 생성된 매장(이름이 "CU 수원역점" 등)을 찾을 수 없었다. `findNearby`는 이름 조건 없이 위도/경도로만 검색한다.

### 프론트엔드: selectCuWithProducts()

```typescript
const selectCuWithProducts = (cu: Mart) => {
    setSelectedCu(cu)
    fetch(`http://localhost:8081/api/marts`, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
            name: cu.name,
            address: cu.address,
            latitude: cu.latitude,
            longitude: cu.longitude
        })
    })
        .then(res => res.json())
        .then((mart: Mart) => fetch(`http://localhost:8081/api/products?martId=${mart.id}`))
        .then(res => res.json())
        .then((data: Product[]) => {
            if (Array.isArray(data)) setProducts(data)
            else setProducts([])
        })
        .catch(() => setProducts([]))
}
```

## 데이터 흐름

```
마커 클릭 (CU 수원역점)
    │
    ▼
POST /api/marts { "name": "CU 수원역점", "latitude": 37.2662, "longitude": 126.9995, "address": "..." }
    │
    ├─ findNearby(37.2662, 126.9995, 0.1) → 100m 이내 기존 매장?
    │   ├─ Yes → 해당 매장 반환
    │   └─ No  → 새 매장 INSERT + catalog 80개 복제 + 랜덤 재고 → 새 매장 반환
    │
    ▼
GET /api/products?martId={id}
    │
    ▼
우측 사이드바에 80개 상품 + 재고 표시
```

## 문제 및 해결

### 문제 1: findNearby가 동적 생성 매장을 못 찾음

`MartRepository.findNearbyCUs()`가 `WHERE m.name = 'CU'` 조건을 사용해서, 이름이 "CU 수원역점"인 동적 생성 매장을 검색하지 못함.

**해결:** 이름 조건이 없는 `findNearby()` 쿼리를 별도로 추가.

### 문제 2: 엔티티에 Setter 없음

초기에는 `@Getter`만 있어서 새 객체 생성 시 필드 설정 불가.

**해결:** `@Setter` 애노테이션과 `import lombok.Setter` 추가.

### 문제 3: 중복 매장 생성

같은 위치를 여러 번 클릭하면 매번 새 매장이 생성됨.

**해결:** 100m(0.1km) 이내 기존 매장이 있으면 생성하지 않고 반환.

---

## 주의사항

- 동적 생성된 매장의 상품 재고는 **랜덤 값** (실제 재고 아님)
- 매번 find-or-create 시 80개의 INSERT가 실행되므로, `@Transactional`로 묶어서 처리
- `POST /api/marts`는 `Map<String, Object>`를 받아 유효성 검사가 없음 (MVP 수준)
- 동적 생성 시점에만 상품이 복제되므로, catalog_product가 업데이트되어도 기존 매장의 상품은 변경되지 않음

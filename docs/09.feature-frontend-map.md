# 09. 프론트엔드: React + Kakao Map 지도 화면

## 개요

React 19 + TypeScript 6 + Vite 8 기반의 SPA. Kakao Map JavaScript SDK를 동적으로 로드하여 지도를 표시하고, 우측 사이드바에 상품 리스트를 렌더링한다.

---

## 기술 스택

| 항목 | 버전 |
|------|------|
| React | 19.2.6 |
| TypeScript | 6.0 |
| Vite | 8.0 |
| Kakao Map SDK | 최신 (동적 로드) |
| HTTP 클라이언트 | 브라우저 내장 `fetch` |

## Kakao Map SDK 동적 로드

```typescript
useEffect(() => {
    const script = document.createElement('script')
    script.src = `//dapi.kakao.com/v2/maps/sdk.js?appkey=${import.meta.env.VITE_KAKAO_MAP_KEY}&autoload=false&libraries=services`
    script.async = true
    document.head.appendChild(script)

    script.onload = () => {
        window.kakao.maps.load(() => {
            const instance = new window.kakao.maps.Map(mapRef.current, {
                center: new window.kakao.maps.LatLng(37.5665, 126.9780),
                level: 3
            })
            setMap(instance)
        })
    }
}, [])
```

- `autoload=false`: SDK 로드 후 수동 초기화
- `libraries=services`: Places API (키워드 검색) 사용
- `declare global { interface Window { kakao: any } }`: TypeScript 타입 선언
- `useRef`로 DOM 요소 참조, `useState`로 map 인스턴스 저장

## 마커 관리

```typescript
const clearMarkers = () => {
    markersRef.current.forEach(m => m.setMap(null))
    markersRef.current = []
}
```

- 모든 마커는 `markersRef` 배열에 저장
- 새 검색 시 `clearMarkers()`로 기존 마커 제거 후 새 마커 추가

## 문제 및 해결

### 문제 1: Kakao SDK TypeScript 오류

`window.kakao`가 TypeScript에서 정의되지 않음.

```typescript
// 해결: 전역 타입 선언
declare global {
    interface Window {
        kakao: any
    }
}
```

### 문제 2: SDK 로드 타이밍 문제

스크립트가 로드되기 전에 `kakao.maps`에 접근하면 에러 발생.

```typescript
// 해결: autoload=false + load() 콜백 패턴
script.onload = () => {
    window.kakao.maps.load(() => {
        // 이 시점에 kakao.maps 사용 가능
    })
}
```

### 문제 3: 이미지 로드 실패

일부 상품 이미지 URL이 유효하지 않아 broken image 아이콘 표시됨.

```tsx
// 해결: onError로 이미지 숨김 처리
<img src={p.imageUrl} alt={p.name}
    onError={e => { (e.target as HTMLImageElement).style.display = 'none' }}
/>
```

### 문제 4: 단일 컴포넌트 집중

모든 UI와 로직이 `App.tsx` 하나에 집중되어 있어 가독성과 유지보수성이 낮음.

→ 향후 개선: `SearchBar`, `MapView`, `ProductList`, `ProductCard` 등으로 분리

### 문제 5: 로딩 상태 없음

API 호출 중 사용자에게 로딩 중임을 알리는 UI가 없음.

→ 향후 개선: `isLoading` 상태 추가, Skeleton UI 도입

## 추가/변경 사항

- Vite 템플릿의 `App.css`와 `assets/` 파일들은 사용되지 않음 (인라인 스타일로 대체)
- `index.css`의 `#root` max-width (1126px)만 적용됨
- Kakao Map API 키는 `.env` 파일에 저장되어 `.gitignore`로 제외됨
- 초기 지도 중심: 서울시청 (37.5665, 126.9780), 레벨 3
- 별도의 CSS 모듈, styled-components, Tailwind CSS 등을 사용하지 않음

# Mart-Compare 프로젝트 작업 내역 정리

## 프로젝트 개요

CU 편의점 행사 상품(1+1, 2+1) 정보를 크롤링하여 데이터베이스에 저장하고, 지도 기반 웹 UI로 조회할 수 있는 풀스택 애플리케이션.

**구성:** Python 크롤러 → MySQL → Spring Boot REST API → React + Kakao Map

---

## Phase 1: CU 행사상품 크롤러 구현 (2026-05-22)

### 작업 내용
- `crawler/cu_crawler.py` — CU 편의점 행사 상품 크롤러 구현
- `crawler/db_insert.py` — 크롤링한 CSV 데이터를 MySQL에 적재하는 스크립트
- `crawler/cu_products.csv` — 수집된 80개 상품 데이터
- `crawler/images/` — 80개 상품 이미지 다운로드

### 접근 방식
- CU 공식 쇼핑몰(`cu.bgfretail.com`)의 `plusAjax.do` 엔드포인트에 POST 요청
- `searchCondition` 파라미터로 1+1(빈값) / 2+1("24") 구분하여 각각 수집
- BeautifulSoup으로 HTML 파싱, 상품명/가격/행사종류/이미지URL 추출
- 수집한 이미지는 `images/` 디렉터리에 인덱스_상품명_파일명 형식으로 저장
- 최종 CSV를 `utf-8-sig` 인코딩으로 저장 (Excel 호환성)

### 문제 및 해결
| 문제 | 원인 | 해결 |
|------|------|------|
| 가격 문자열에 쉼표 포함 | `"4,500"` 형태의 문자열 | `int(str(price).replace(",", ""))`로 변환 |
| 상품명에 파일명으로 쓸 수 없는 문자 포함 | 윈도우/리눅스 제한 문자 | 정규식 `re.sub(r'[\\/*?:"<>|]', "", name)`으로 제거 |
| 이미지 URL이 `//`로 시작 | 프로토콜 상대 경로 | `"https:" + src`로 보완 |

### 주요 결정 사항
- Selenium 대신 requests + BeautifulSoup 선택 (CSR이 아닌 SSR 페이지이므로 가능)
- 띄어쓰기 포함 컬럼명(예: `상품명`)을 그대로 CSV 헤더로 사용 (가독성 우선)
- `utf-8-sig` 인코딩 사용 (한글 Excel에서 열 때 깨짐 방지)

## Phase 2: DB 스키마 설계 및 데이터 적재 (2026-05-22 ~ 2026-05-23)

### 작업 내용
- `mysql/mart, product table create.sql` — MySQL 스키마 정의
  - `mart` 테이블: 편의점 지점 정보 (id, name, address, latitude, longitude)
  - `product` 테이블: 행사 상품 정보 (id, mart_id(FK), name, price, event_type, image_url)
- `mart` 시드 데이터: CU 기본 정보 INSERT

### 접근 방식
- `ddl-auto=none` 전략 → JPA의 자동 DDL 생성 기능을 사용하지 않고, 수동 SQL로 스키마를 관리
- UTF-8MB4 인코딩 사용 (이모지 등 4바이트 문자 대응)
- `mart` 테이블에 위도/경도 컬럼을 미리 설계 (향후 지도 기능 확장 고려)

### 문제 및 해결
- `product` 테이블에 이미지 URL 컬럼을 나중에 추가해야 했음 → `ALTER TABLE`로 대응
- CSV의 컬럼명과 DB 컬럼명이 달라 매핑 필요 → `db_insert.py`에서 직접 매핑

## Phase 3: Spring Boot 백엔드 구현 (2026-05-23)

### 작업 내용
- `backend/` — Spring Boot 3.5.14 + Java 17 + Gradle 프로젝트
- `entity/Product.java` — JPA 엔티티 (상품명, 가격, 행사종류, 이미지URL)
- `repository/ProductRepository.java` — JPA 리포지토리
- `service/ProductService.java` — 비즈니스 로직 레이어
- `controller/ProductController.java` — REST API (`GET /api/products`)
- `config/CorsConfig.java` — CORS 설정 (localhost:5173 허용)
- `application.properties` — MySQL 연결, 포트 8081, JPA 설정

### 접근 방식
- **레이어드 아키텍처:** Controller → Service → Repository → Entity
- **Lombok 활용:** `@RequiredArgsConstructor`, `@Getter`로 보일러플레이트 최소화
- **읽기 전용 엔티티:** `@Getter`만 선언하고 `@Setter` 없이 사용 (현재는 조회만 필요)
- **CORS 설정:** Vite 개발 서버(`localhost:5173`)의 API 호출을 허용하도록 `WebMvcConfigurer` 구현

### 문제 및 해결
| 문제 | 해결 |
|------|------|
| MySQL connector 의존성 누락 | `runtimeOnly 'mysql:mysql-connector-j'` 추가 |
| CORS 차단 (프론트에서 API 호출 실패) | `CorsConfig`에서 `allowedOrigins("http://localhost:5173")` 설정 |
| Gradle Java 버전 불일치 | `gradle.properties`에 `org.gradle.java.home` 명시 |

### 주요 결정 사항
- Spring Boot 3.5.14 (최신 버전) 채택
- JPA `ddl-auto=none`으로 수동 스키마 관리
- DTO 없이 엔티티를 직접 응답으로 반환 (MVP 단계에서는 단순성 우선)
- 서버 포트 8081 사용 (React dev 서버 5173과 충돌 방지)

## Phase 4: React + Kakao Map 프론트엔드 구현 (2026-05-24 ~ 2026-05-25)

### 작업 내용
- **2026-05-24:** `frontend/` React + Vite + TypeScript 프로젝트 초기화 및 Kakao Map 연동
  - Vite 8 + React 19 + TypeScript 6 프로젝트 생성
  - `App.tsx`: Kakao Map SDK 동적 로드, 서울시청 중심 지도 표시, CU 마커 1개
  - 마커 클릭 시 InfoWindow 표시

- **2026-05-25 오전:** Spring Boot API 연동 및 상품 리스트 화면 구현
  - `fetch('http://localhost:8081/api/products')`로 API 호출
  - 우측 사이드바: 300px 너비, 스크롤 가능한 상품 리스트
  - 상품 이미지(80x80), 이름(굵게), 가격(toLocaleString), 행사종류(빨간색) 표시
  - `onError`로 이미지 로드 실패 시 숨김 처리

- **2026-05-25:** Kakao Map 마커 개선
  - 커밋 `ea634cc`에서 마커 동작 개선

### 접근 방식
- **Kakao Map SDK 동적 로드:** `autoload=false` 옵션으로 스크립트를 동적으로 삽입 후 `kakao.maps.load()`로 초기화
- **인라인 스타일:** 별도 CSS 파일 없이 `style={{}}` 방식 사용 (컴포넌트 규모가 작아 오버헤드 최소화)
- **이미지 에러 핸들링:** `onError` 이벤트로 이미지 DOM을 `display: none` 처리 (이미지 URL이 유효하지 않은 경우 대응)

### 문제 및 해결
| 문제 | 해결 |
|------|------|
| Kakao Maps API 키 노출 우려 | `.env` 파일에 `VITE_KAKAO_MAP_KEY` 저장, `.gitignore`에 `.env` 등록 |
| Kakao SDK `kakao is not defined` | `autoload=false` + `kakao.maps.load()` 콜백 패턴 사용 |
| CORS 오류 (브라우저 차단) | 백엔드 `CorsConfig`에서 `localhost:5173` 허용 |
| `window.kakao` 타입 에러 | `declare global { interface Window { kakao: any } }`로 타입 선언 |
| 이미지 URL이 DB에 없는 경우 | `onError` 핸들러로 이미지 태그 숨김 |
| 가격 숫자 포맷팅 | `toLocaleString()`으로 3자리마다 쉼표 표시 |

### 주요 결정 사항
- React 19 + Vite 8 최신 스택 채택 (2026년 기준)
- TypeScript 6 사용 (strict 모드)
- Kakao Map API 키를 Vite의 `import.meta.env`로 접근
- 지도는 좌측(flex:1), 상품 리스트는 우측(300px 고정)의 가로 분할 레이아웃
- 마커는 하드코딩된 1개 (CU 서울시청점) — 현재 DB에 좌표 데이터가 없으므로 임시

## Phase 5: WSL 환경 작업본 업로드 (2026-05-27)

### 작업 내용
- Windows 환경에서 작업한 내용을 WSL 환경으로 이전
- `backend/.gradle/`, `backend/.idea/` 등 IDE/빌드 캐시 포함하여 전체 업로드
- Git LFS 관련 `.gitattributes` 설정 확인

### 문제 및 해결
- **문제:** Windows에서 생성된 `:Zone.Identifier` alternate data stream 파일들이 함께 트래킹됨
  - `backend/.gitattributes:Zone.Identifier`, `backend/src/main/java/.../BackendApplication.java:Zone.Identifier` 등
  - 이는 Windows에서 파일을 다운로드할 때 NTFS가 자동 생성하는 보안 메타데이터
  - **현재 상태:** Git에 포함되어 있으나 기능에 영향을 주지 않음. 추후 `.gitignore`에 `*.java:Zone.Identifier` 패턴으로 제거 가능
- WSL 환경에서 빌드/실행 시 Gradle wrapper의 `gradle.properties`에 Java 경로 명시 필요

## 현재까지의 전체 데이터 흐름

```
[CU 쇼핑몰 웹사이트]
       │
       ▼ (HTTP POST + HTML 파싱)
cu_crawler.py ──▶ cu_products.csv ──▶ db_insert.py
                                              │
                                              ▼ (JDBC INSERT)
                                          [MySQL]
                                     database: mart_compare
                                     table: product (80 rows)
                                              │
                                              ▼ (JPA SELECT)
                              Spring Boot /api/products
                                              │
                                              ▼ (HTTP JSON)
                  React App ──┬── Kakao Map (좌측)
                              └── Product List (우측, 80개)
```
